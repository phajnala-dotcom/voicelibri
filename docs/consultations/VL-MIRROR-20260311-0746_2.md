Subject: Implementácia CLAP modelu s Anti-Training Auditným Protokolom

1. Zhrnutie diskusie
Hlavným cieľom je integrácia open-source modelu LAION-CLAP na automatizované generovanie sémantických popisov a embeddingov pre zvukové vzorky: 1. ktoré takéto popisy doposiaľ nemajú; 2. vygenerovanie sekundárnych / alternatívnych popisov, ake je zvuk použiteľný aj pre inú entitu/dej/jav, čím by sa zvýšila využiteľnosť databázy a možnosť dosiahnutia lepšej koínovej zhody. Kľúčovou prioritou je zabezpečenie právnej preukázateľnosti, že model sa na týchto dátach netrénuje, čím sa chránia licenčné podmienky dodávateľov (napr. Free To Use Sounds). Proces bude prebiehať výhradne lokálne (on-device/local server) bez odosielania dát do cloudu.

2. Návrhy na implementáciu
2.1 Lokálna Inference Architektúra
Typ: Infrastructure / ML

Priorita: Kritická

Implementácia:

Local-First Engine: Model beží v izolovanom prostredí. Dáta sú načítavané len do dočasnej pamäte RAM pre potreby analýzy.

Zero-Shot Pipeline: Využitie predtrénovaných váh LAION-CLAP na klasifikáciu bez nutnosti akéhokoľvek dolaďovania (Fine-tuning).

No-Cloud Policy: Striktná blokácia sieťovej komunikácie počas behu analytického skriptu.

2.2 Anti-Training Auditný Modul (Právna ochrana)
Typ: Compliance / Security

Priorita: Ultimatívna

Technická špecifikácia:

Immutable Loguru Logs: Automatické generovanie JSON logov, ktoré obsahujú SHA-256 hash modelu. Identický hash pred a po analýze preukazuje, že váhy modelu neboli zmenené (žiadny tréning).

Torch No-Grad Lock: Implementácia torch.set_grad_enabled(False) na úrovni jadra skriptu, čo technicky znemožňuje spustenie tréningového algoritmu.

Telemetry Artifacts: Systémový záznam o trvaní procesu a využití CPU/GPU, ktorý korešponduje s čistou inferenciou, nie s výpočtovo náročným tréningom.

Dotknuté súbory: packages/analysis-engine/audit_logger.py, scripts/run_clap_analysis.py.

2.3 VS Code Proof-of-Process
Typ: Development Workflow

Priorita: Vysoká

Mechanizmus:

Activity Trace: Záznam aktivity cez VS Code rozšírenie (napr. Activity Tracker) ako sekundárny dôkaz o charaktere práce na kóde.

Git Commit Signatures: Každá zmena v analytickom skripte musí byť podpísaná, aby bola jasná história integrity auditných mechanizmov.

3. Akčný plán (Roadmap pre PoC)
Model Setup: Stiahnutie LAION-CLAP modelu a verifikácia jeho kontrolného súčtu (Hash).

Audit Core: Vytvorenie pomocnej knižnice pre generovanie nepozmeniteľných logov.

Inference Test: Spustenie analýzy na vzorke 10 zvukov a vygenerovanie prvého Protokolu o spracovaní.

Validation: Manuálna kontrola sémantickej presnosti popisov voči auditným záznamom.

Documentation: Príprava finálneho reportu pre dodávateľa zvukovej databázy ako dôkaz o dodržaní licencií.

4. Technické poznámky
Licenčná zhoda: Celý proces je navrhnutý tak, aby bol "Audit-Ready" pre prípadnú kontrolu zo strany majiteľov autorských práv.

Integrity Check: Odporúča sa pravidelne ukladať screenshoty bežiaceho systému ako doplnok k digitálnym logom.
